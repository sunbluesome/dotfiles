# インタビューガイド

プロジェクト憲章作成のためのインタビューテクニック。

## 基本原則

### 1. オープンクエスチョンから始める

**良い例**:
- 「このプロジェクトについて教えてください」
- 「どのような課題を解決したいですか？」

**避けるべき例**:
- 「予算は1000万円ですか？」（クローズド）
- 「Pythonで作りますよね？」（誘導）

### 2. 5W1Hで深掘り

| 質問 | 目的 |
|------|------|
| Why | 動機・背景の理解 |
| What | 成果物・スコープの明確化 |
| Who | ステークホルダーの特定 |
| When | 期限・マイルストーンの把握 |
| Where | 環境・制約の理解 |
| How | アプローチ・手法の確認 |

### 3. 沈黙を恐れない

ユーザーが考える時間を与える。すぐに次の質問に移らない。

## フェーズ別テクニック

### Phase 1: Why（なぜ）

**目標**: プロジェクトの存在意義を明確にする

**効果的な質問**:
- 「このプロジェクトがなかったら何が困りますか？」
- 「誰が一番このプロジェクトを必要としていますか？」
- 「半年後、このプロジェクトが成功したらどうなっていますか？」

**ありがちな落とし穴**:
- 「〜を作りたい」という What から始まってしまう
- Why が曖昧なままスコープを決めてしまう

**対処法**:
- 「それを作ると、何が解決しますか？」と深掘り
- 「なぜ今、このタイミングで必要ですか？」

### Phase 2: What（何を）

**目標**: 具体的な成果物とスコープを定義する

**効果的な質問**:
- 「プロジェクト完了時に何が手に入りますか？」
- 「最低限これだけは必要、というものは何ですか？」
- 「逆に、今回はやらないと決めていることはありますか？」

**スコープ膨張を防ぐ**:
- 「それは Must ですか、Nice-to-have ですか？」
- 「MVPに必要な機能だけ挙げるとしたら？」

**成功指標を引き出す**:
- 「成功したかどうか、どうやって判断しますか？」
- 「数値で測れる目標はありますか？」

### Phase 3: How（どうやって）

**目標**: アプローチと制約を明確にする

**効果的な質問**:
- 「技術的な制約はありますか？」
- 「既存システムとの連携は必要ですか？」
- 「チームの得意な技術はありますか？」

**制約を引き出す**:
- 「絶対に使わなければならない技術はありますか？」
- 「使ってはいけない技術はありますか？」
- 「セキュリティやコンプライアンスの要件は？」

**タイムラインを確認**:
- 「期限は決まっていますか？」
- 「外部要因で動かせないマイルストーンは？」

## 困ったときの対処法

### ユーザーが答えられない場合

**原因1**: 質問が抽象的すぎる
- → 具体例を提示する
- → 選択肢を提示する（AskUserQuestionのoptions活用）

**原因2**: まだ決まっていない
- → 「仮でいいので」と前置き
- → 「TBD」を許容し、後で更新

**原因3**: 専門用語が伝わっていない
- → 平易な言葉で言い換え
- → 例を使って説明

### 情報が多すぎる場合

- 「最も重要な3つに絞るとしたら？」
- 「優先順位をつけるとしたら？」
- 構造化して整理し、確認を取る

### 矛盾がある場合

- 「先ほど〇〇とおっしゃっていましたが、△△とはどう整合しますか？」
- 矛盾を指摘するのではなく、理解を深める姿勢で

## AskUserQuestion活用パターン

### パターン1: 二者択一

```python
AskUserQuestion(
  questions=[{
    "question": "開発スピードと品質、どちらを優先しますか？",
    "header": "優先度",
    "options": [
      {"label": "スピード優先", "description": "早くリリースして改善"},
      {"label": "品質優先", "description": "十分なテストを経てリリース"}
    ],
    "multiSelect": False
  }]
)
```

### パターン2: 複数選択

```python
AskUserQuestion(
  questions=[{
    "question": "このプロジェクトで重視する非機能要件は？",
    "header": "要件",
    "options": [
      {"label": "性能", "description": "レスポンス時間、スループット"},
      {"label": "可用性", "description": "稼働率、障害対応"},
      {"label": "セキュリティ", "description": "認証、暗号化"},
      {"label": "保守性", "description": "コードの読みやすさ、拡張性"}
    ],
    "multiSelect": True
  }]
)
```

### パターン3: 段階的深掘り

最初の質問でカテゴリを特定し、次の質問で詳細を聞く。

```python

    "header": "タイプ",

      {"label": "インフラ", "description": "基盤・自動化・運用"}
    ]
  }]
)

# Step 2: タイプに応じた詳細質問
# （回答に応じて異なる質問セットを使用）
```

## チェックリスト

インタビュー完了時に確認:

### Why
- [ ] 課題・背景が明確
- [ ] 目的が1文で言える
- [ ] 成功イメージがある

### What
- [ ] 成果物リストがある
- [ ] KPI/成功指標がある
- [ ] スコープ（In/Out）が明確

### How
- [ ] アプローチが決まっている
- [ ] 技術選定の方針がある
- [ ] 制約条件が把握できている

### その他
- [ ] ステークホルダーが特定できている
- [ ] 主要リスクが把握できている
- [ ] 不明点が「TBD」として記録されている
