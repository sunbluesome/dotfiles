# 専門用語の言い換え辞書

DS以外の部署向けに専門用語を分かりやすく言い換えるための辞書。

## 精度指標

| 専門用語 | 言い換え | 補足説明 |
|----------|----------|----------|
| WMAPE | 予測と実績のずれ（%） | 「全体で何%ずれているか」の指標 |
| Bias | 予測の偏り | 過大予測なら正、過小予測なら負 |
| MAPE | 商品ごとの平均誤差率 | 各商品を均等に評価した誤差 |
| sMAPE | 対称平均誤差率 | 予測・実績の大きさに依存しない誤差 |
| RMSE | 二乗誤差の平均の平方根 | 大きな誤差を重く評価する指標 |
| MAE | 平均絶対誤差 | 誤差の絶対値の平均 |
| R² | 決定係数 | モデルの説明力（1に近いほど良い） |

## インフラ・システム

| 専門用語 | 言い換え | 補足説明 |
|----------|----------|----------|
| パイプライン | 自動処理の仕組み | データを順番に処理する流れ |
| Docker | 実行環境のパッケージ | どこでも同じ環境で動かせる仕組み |
| ECR | Dockerイメージの保管庫 | AWSのコンテナ保管サービス |
| SageMaker | AWS上の計算サービス | 機械学習用のクラウド計算環境 |
| StepFunctions | 処理の自動実行管理 | ジョブを順番に実行するサービス |
| CloudWatch | 監視サービス | メトリクスやアラートを管理 |
| CI/CD | 自動テスト・デプロイ | コード変更時に自動でテスト・公開 |

## 開発用語

| 専門用語 | 言い換え | 補足説明 |
|----------|----------|----------|
| DTO | データの型定義 | データの構造を定義したもの |
| リファクタリング | コードの整理・改善 | 動作を変えずに内部構造を改善 |
| テストカバレッジ | テストの網羅率 | コードのどれだけがテストされているか |
| Protocol | インターフェース定義 | 部品の接続仕様 |
| DI（依存性注入） | 部品の差し替え可能な設計 | テストや拡張が容易になる設計手法 |

## 機械学習

| 専門用語 | 言い換え | 補足説明 |
|----------|----------|----------|
| モデル | 予測の仕組み | データから予測を行うプログラム |
| フィッティング | モデルの学習 | データに合わせて調整すること |
| 過学習 | 学習データに過度に適合 | 新しいデータで精度が落ちる状態 |
| 特徴量 | 予測に使う情報 | モデルに入力するデータ項目 |
| ハイパーパラメータ | 調整パラメータ | モデルの動作を制御する設定値 |
| 階層ベイズ | グループ間で情報を共有する手法 | データが少ない商品でも安定した推定が可能 |

## データ処理

| 専門用語 | 言い換え | 補足説明 |
|----------|----------|----------|
| 前処理 | データの整理・加工 | 分析前にデータを使える状態にする |
| 正規化 | データの尺度を揃える | 異なる単位のデータを比較可能に |
| 欠損値 | 抜けているデータ | データが存在しない箇所 |
| 外れ値 | 異常に大きい/小さい値 | 通常の範囲から外れたデータ |
| 集約 | データをまとめる | 日別→週別など、粒度を変える |

## 文章表現のコツ

### 数値は具体的に

❌ 「精度が改善した」
✅ 「予測誤差が10%から5%に半減した」

### 専門用語は括弧で補足

❌ 「WMAPEを計算するパイプラインを構築」
✅ 「予測誤差（WMAPE）を自動計算する仕組みを構築」

### 比喩を使う

❌ 「階層ベイズモデルを適用」
✅ 「似た商品の情報を借りてくることで、データが少ない商品でも安定した予測が可能に」

### 結果から書く

❌ 「問題Aがあり、手法Bを試し、結果Cになった」
✅ 「結果Cを達成。問題Aに対して手法Bが有効だった」
